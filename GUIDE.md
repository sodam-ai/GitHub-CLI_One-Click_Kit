## 1. 들어가기 전에

**GitHub** 은 코드(프로그램 파일)를 인터넷에 저장하고, 여러 사람이 함께 작업하는 공간입니다.
**GitHub CLI**(`gh`)는 GitHub가 만든 **공식 프로그램**으로, 웹사이트 대신 컴퓨터에서
**명령어**로 GitHub를 다룹니다.

문제는, `gh` 를 쓰려면 보통 **영어 명령어를 직접 외워서 입력**해야 한다는 점입니다.
이 키트는 그 부분을 **한국어 메뉴**로 바꿔, **번호만 고르면** 되도록 만들었습니다.

이 키트는 **3개의 실행 파일**로 이루어져 있습니다.

| 파일 | 역할 | 언제 쓰나요 |
|---|---|---|
| `INSTALL.bat` | 설치 | 맨 처음 한 번 |
| `RUN.bat` | 사용 | GitHub 를 쓸 때마다 |
| `UNINSTALL.bat` | 제거 | 더 이상 안 쓸 때 |

## 2. 사전 준비물 / 필요 프로그램

| 항목 | 내용 | 꼭 필요? |
|---|---|---|
| 운영체제 | Windows 10 (22H2 이상) 또는 Windows 11 | 필수 |
| 인터넷 | 설치·로그인·대부분 기능에 필요 | 필수 |
| GitHub 계정 | https://github.com 에서 무료 가입 | 필수 |
| winget | Windows 10 22H2+·11 에 기본 포함된 설치 도구 | 셋 중 하나면 됨 |
| Scoop | 선택 설치 도구 (https://scoop.sh) | 셋 중 하나면 됨 |
| (둘 다 없음) | 키트가 공식 `.msi` 를 직접 받아 설치 | 셋 중 하나면 됨 |

> **정리:** 미리 깔아야 할 프로그램은 **없습니다.** Windows·인터넷·GitHub 계정만 있으면
> `INSTALL.bat` 이 나머지를 알아서 합니다.

**winget 이 있는지 확인하는 법 (선택):** 시작 메뉴에서 `cmd` 를 열고 `winget --version`
을 입력해 보세요. 버전 번호가 나오면 있는 것입니다. 없어도 키트가 알아서 처리하니 걱정하지
않아도 됩니다.

## 3. 다운로드 방법

1. 이 키트가 담긴 **폴더 또는 ZIP 파일**을 받습니다.
2. **ZIP 으로 받았다면 꼭 압축을 푸세요.**
   - ZIP 파일을 마우스 **우클릭 → 압축 풀기(또는 "모두 압축 해제")** 를 누릅니다.
   - ZIP 안에서 바로 더블클릭하면 일부 기능이 동작하지 않을 수 있습니다.
3. 푼 폴더 안에 아래 3개가 보이면 준비 완료입니다.
   - `INSTALL.bat` · `RUN.bat` · `UNINSTALL.bat`

> **참고 — 차단 해제:** 인터넷에서 받은 ZIP 은 Windows 가 "차단" 표시를 해 둘 수 있습니다.
> ZIP 파일을 우클릭 → **속성** → 아래쪽 **[차단 해제] 체크 → 확인** 후 압축을 풀면
> SmartScreen 경고가 줄어듭니다. (안 해도 됩니다.)

## 4. 설치 방법 (자세히)

### 4.1 기본 — 이렇게만 하면 됩니다

1. `INSTALL.bat` 을 **더블클릭**합니다.
2. 검은색 창이 열리고 자동으로 진행됩니다. 화면 안내를 따라가기만 하면 됩니다.
3. 끝에 **"지금 바로 시작할까요? [Y/n]"** 이 나오면 **Y** 를 누르세요. `RUN.bat` 이 자동으로 열립니다.

> **"Windows가 PC를 보호했습니다"(SmartScreen)** 창이 뜨면 → **추가 정보 → 실행**을 누르세요.

### 4.2 설치 중 무슨 일이 일어나나요 (5단계)

`INSTALL.bat` 은 아래 순서로 동작합니다. 각 단계가 화면에 표시됩니다.

| 단계 | 하는 일 |
|---|---|
| 이미 설치 확인 | gh 가 이미 있으면 "업그레이드할까요?" 를 물어봅니다. |
| 관리자 권한 확인 | 권한이 없으면 [1] 권한 올려 다시 시작 / [2] 그냥 계속 을 물어봅니다. |
| 인터넷 확인 | 연결이 안 되면 경고만 하고(차단하지 않음) 계속할 수 있습니다. |
| [2/5] 설치 도구 확인 | winget 또는 Scoop 이 있는지 봅니다. |
| [3/5] 설치 | winget/Scoop 으로 설치하거나, 둘 다 없으면 공식 `.msi` 를 직접 받습니다. |
| [4/5] 경로 새로고침 | 방금 설치한 gh 를 현재 창에서 찾도록 PATH 를 갱신합니다. |
| [5/5] 확인 | gh 버전을 출력해 설치를 확인합니다. |

### 4.3 관리자 권한이 필요할 때

GitHub CLI 설치는 보통 **관리자 권한**이 있어야 끝까지 됩니다. 권한이 없으면 `INSTALL.bat`
이 다음을 물어봅니다.

- **[1] 관리자 권한으로 다시 시작 (권장)** — 선택하면 Windows **허용 창(UAC)** 이 뜹니다.
  여기서 **[예]** 를 누르면 권한이 있는 상태로 다시 실행됩니다.
- **[2] 그냥 계속하기** — 권한 없이 진행합니다. (실패하면 [1] 로 다시 시도하세요.)

> 만약 [1] 을 골랐는데 아무 일도 없으면, 허용 창에서 [아니오]를 누른 것일 수 있습니다.
> `INSTALL.bat` 을 다시 더블클릭해서 [1] 을 다시 고르거나, 파일을 **우클릭 → 관리자 권한으로
> 실행**하세요.

### 4.4 SmartScreen 경고가 떴어요

인터넷에서 받은 `.bat` 은 처음 실행할 때 Windows 가 보호 창을 띄울 수 있습니다.
이것은 위험 신호가 아니라 **출처 미확인 파일에 대한 일반 안내**입니다.

→ **추가 정보** 글자를 누른 뒤 → **실행** 버튼을 누르면 됩니다.

### 4.5 winget·Scoop 이 둘 다 없을 때 (자동 다운로드)

설치 도구가 하나도 없으면 `INSTALL.bat` 이 묻습니다: **"자동으로 받아서 설치할까요? [Y/n]"**

- **Y** 를 누르면 → 컴퓨터 종류(amd64/arm64)에 맞는 **공식 설치 파일(.msi)** 을 GitHub
  공식 서버에서 받아 자동 설치합니다.
- 자동 받기가 실패하면 → 공식 다운로드 페이지가 브라우저로 열립니다. 거기서
  파일 이름에 `windows_amd64.msi` 가 들어간 파일을 받아 **더블클릭**하면 됩니다.
  (공식 페이지: https://github.com/cli/cli/releases/latest )

### 4.6 설치가 잘 됐는지 확인

설치가 끝나면 화면에 `gh version 2.x.x ...` 같은 줄과 **"설치가 확인되었습니다 [확인됨]"**
이 보입니다. 만약 **"현재 창에서는 gh 를 찾지 못했습니다"** 가 나오면, 터미널 창이나 PC 를
**다시 시작**하면 반영됩니다.

## 5. 실행 방법

1. `RUN.bat` 을 **더블클릭**합니다.
2. 맨 위에 현재 gh 버전과 **로그인 상태**가 표시되고, 그 아래 **17개 메뉴**가 나옵니다.
3. 하고 싶은 기능의 **번호를 입력하고 Enter** 를 누릅니다.
4. 각 작업이 끝나면 "아무 키나 누르세요" 가 나옵니다. 키를 누르면 메뉴로 돌아갑니다.
5. 끝낼 때는 **`0`** 을 눌러 종료합니다.

> 잘못된 번호(예: 99)를 입력하면 **"0 부터 17 사이의 번호를 입력해 주세요"** 안내가 나옵니다.

## 6. 빠른 시작 — 쉬운 시작 마법사

처음이라면 16개 메뉴를 다 볼 필요 없이 **`[17] 쉬운 시작`** 하나면 됩니다.

1. **[1단계] 로그인 확인** — 로그인이 안 되어 있으면 지금 할지 물어봅니다. (브라우저 로그인)
2. **[2단계] 무엇을 할까요?** — 셋 중 고릅니다.
   - `[1] 남의 코드를 내 PC 로 받기` (clone)
   - `[2] 내 저장소 새로 만들기`
   - `[3] 지금은 둘러보기만`
3. **끝!** 이후에는 메인 메뉴에서 더 많은 기능을 쓰면 됩니다.

## 7. 로그인 방법

대부분의 기능은 **로그인이 필요**합니다. 메뉴 `[1] 로그인 관리` 에서 할 수 있습니다.

**가장 쉬운 방법 — 브라우저 로그인 (`[1] 로그인 관리` → `[1] 로그인 - 브라우저`)**

1. 잠시 후 화면에 **8자리 코드**(예: `ABCD-1234`)가 나옵니다.
2. 그 코드를 **복사**하거나 적어 둡니다.
3. **자동으로 열리는 브라우저**(또는 안내된 주소)에 그 코드를 **붙여넣습니다.**
4. **Authorize(승인)** 버튼을 누르면 로그인이 끝납니다.
5. 다시 메뉴로 돌아오면 위쪽 상태가 **"로그인됨"** 으로 바뀝니다.

**다른 로그인 방법**

- `[2] 로그인 - 토큰` : GitHub 에서 만든 토큰 문자열로 로그인. (토큰 발급:
  https://github.com/settings/tokens )
- `[3] 로그인 - Enterprise` : 회사용 GitHub 서버 주소(호스트)를 입력해 로그인.
- `[4] 로그인 상태 확인` / `[5] 로그아웃` / `[6] 토큰 보기` / `[7] 토큰 새로고침` / `[8] 계정 전환`

## 8. 사용·작동 방법 — 메뉴별 안내

각 메뉴는 번호를 고르면 세부 항목이 나옵니다. 입력이 필요한 항목은 화면이 물어봅니다.

### [1] 로그인 관리
로그인/로그아웃, 상태 확인, 토큰 보기·새로고침, 계정 전환. (위 [7. 로그인 방법] 참고)

### [2] 저장소 작업 (Repository)
코드 저장소를 만들고 복제하고 관리합니다.
- 저장소 복제(clone), 새로 만들기, 내 저장소 목록, 정보 보기, 포크, 이름 변경,
  **삭제(영구·`YES` 확인)**, 브라우저에서 열기, 내 활동, 보관(archive).
- 복제·포크·이름변경 등은 `owner/repo`(예: `cli/cli`) 형식이나 주소를 입력합니다.

### [3] 풀 리퀘스트 (Pull Request)
"이렇게 코드를 바꾸자"는 제안입니다.
- 목록, 만들기, 보기, 체크아웃, **병합(merge)**, 닫기, 다시 열기, 변경내용(diff),
  상태, 검토 준비완료, 리뷰, 검사(checks). 대부분 **PR 번호**를 입력합니다.

### [4] 이슈 작업 (Issue)
할 일·버그·문의를 기록합니다.
- 목록, 만들기, 보기, 닫기, 다시 열기, 수정, 댓글, 상태, 고정(pin), 이동(transfer),
  라벨 관리(목록/만들기/**삭제 `YES`**). 대부분 **이슈 번호**를 입력합니다.

### [5] Gist
코드 조각이나 메모를 빠르게 공유합니다.
- 내 목록, 만들기(비공개/공개), 보기, 수정, **삭제(`YES`)**, 복제.

### [6] 릴리스 (Release)
배포용 버전을 만듭니다.
- 목록, 만들기(태그 입력, 예 `v1.0.0`), 보기, 내려받기, 수정, **삭제(`YES`)**.

### [7] 워크플로 / Actions
GitHub 의 자동화 작업 실행을 봅니다.
- 실행 목록/보기/실시간 보기/실패만 재실행/취소, 워크플로 목록/실행, 결과물 내려받기, 로그 보기.

### [8] SSH / GPG 키
안전 접속용(SSH)·서명용(GPG) 키를 관리합니다.
- 각각 목록/추가/**삭제(`YES`·되돌릴 수 없음)**.

### [9] 확장 기능 (Extension)
gh 에 기능을 더하는 확장 프로그램.
- 목록, 설치, 업그레이드, 전체 업그레이드, 제거, 만들기, 둘러보기.

### [10] 설정 / 별칭 (Config / Alias)
환경설정과 자주 쓰는 명령 단축키.
- 설정 목록/보기/변경 (예: `editor`, `git_protocol`, `browser`, `pager`).
- 별칭 목록/만들기/삭제/가져오기.

### [11] GitHub 검색 (Search)
- 저장소 / 이슈 / PR / 코드 / 커밋 검색. 검색어를 입력합니다.

### [12] Codespace
인터넷에 있는 클라우드 개발 환경.
- 목록, 만들기, SSH 접속, VS Code 로 열기, 중지, 삭제, 로그.

### [13] API 호출 (고급)
GitHub API 를 직접 호출합니다. (개발자용)
- GET / POST 요청, 내 정보 보기(`/user`), 사용량 제한(`/rate_limit`), GraphQL 질의.

### [14] 버전 확인 / 업데이트
- 설치된 gh 버전 확인, winget/Scoop 으로 업데이트, 자세한 버전 정보.

### [15] 직접 명령 입력 (고급)
- `gh` 를 뺀 나머지 명령을 직접 입력합니다. 예: `repo list --limit 10`.
- 메인 메뉴로 돌아가려면 `back` 을 입력합니다.

### [16] 문제 해결

## 11. 제거 방법

1. `UNINSTALL.bat` 을 **더블클릭**합니다.
2. gh 설정 파일이 있으면 **"설정을 백업할까요? [y/N]"** 을 물어봅니다.
   - `y` 를 누르면 키트 폴더 안 `gh-config-backup` 폴더에 백업합니다.
3. **경고**가 표시되고, **"제거하려면 YES 를 입력하세요"** 가 나옵니다.
   - `YES` 를 입력해야만 진행됩니다. (대문자)
4. 순서대로: GitHub 로그아웃 → gh 제거(winget/Scoop) → 설정·캐시 정리 → 제거 확인.

> 제거 후에도 키트 폴더(.bat·문서)는 남습니다. 필요 없으면 폴더를 휴지통으로 옮기세요.
> 백업한 `gh-config-backup` 폴더는 원하면 따로 보관하거나 삭제하면 됩니다.

## 12. 문제·오류 대처

**먼저 시도:** `RUN.bat` → **`[16] 문제 해결`** (자동 점검 + 한국어 처방).

| 증상 | 원인(가능성) | 해결 |
|---|---|---|
| `gh` 를 찾을 수 없다 | 설치 직후 경로 미반영 | 터미널/PC 다시 시작 |
| 설치가 실패한다 | 관리자 권한 부족 | `INSTALL.bat` 우클릭 → 관리자 권한으로 실행 |
| winget/scoop 없음 | 설치 도구 미설치 | INSTALL 의 자동 `.msi` 받기(Y) 또는 공식 페이지에서 직접 받기 |
| 영어 오류가 뜬다 | 인터넷 끊김/방화벽 | 와이파이·랜선 확인, 회사망이면 방화벽 확인 |
| 한글이 깨져 보인다 | 파일이 ANSI/CP949 로 저장됨 | 원본을 다시 받아 사용(편집 시 UTF-8 유지) |
| SmartScreen 경고 | 인터넷에서 받은 파일 | 추가 정보 → 실행 |
| 로그인이 안 끝난다 | 코드 미입력/브라우저 안 열림 | 8자리 코드를 브라우저에 붙여넣고 Authorize |
| 대부분 기능이 막힘 | 로그인 안 됨 | `[1] 로그인 관리` 에서 로그인 |

**도움을 요청할 때:** `[16] 문제 해결` 에서 결과를 `gh-help-log.txt` 로 저장해 함께 보여주면
원인 파악이 빠릅니다.

## 14. 자주 묻는 질문 (FAQ)

**Q. 무료인가요?**
A. 키트도, GitHub CLI 도 무료입니다. GitHub 계정도 기본 무료입니다.

**Q. 인터넷이 꼭 필요한가요?**
A. 네. 설치·로그인·검색·저장소 작업 등 거의 모든 기능이 인터넷을 씁니다.

**Q. 안전한가요? 비밀번호가 새나가지 않나요?**
A. 로그인은 GitHub **공식 절차**(`gh`)를 그대로 쓰며, 키트가 비밀번호를 따로 저장하지 않습니다.
모든 작업은 **사용자 컴퓨터에서** 이루어집니다.

**Q. 실수로 저장소를 지울까 봐 걱정돼요.**
A. 삭제·제거 같은 위험 작업은 **`YES` 를 직접 입력**해야만 진행됩니다.

**Q. macOS / Linux 에서도 되나요?**
A. 이 키트의 `.bat` 파일은 **Windows 전용**입니다.

## 15. 개발자 노트

- **인코딩:** `.bat` 파일은 **UTF-8(BOM 없음)** + 첫 줄 `chcp 65001` 조합으로 한글을 표시합니다.
  메모장 등에서 **ANSI/CP949 로 다시 저장하면 한글이 모두 깨집니다.** 편집은 반드시 **UTF-8** 로.
- **구조:** 메뉴는 라벨(`:MENU`)과 `goto` 로 구성된 표준 배치 스크립트입니다.
- **설치 폴백:** winget → Scoop → 공식 `.msi`(PowerShell 다운로드) 순으로 시도합니다.
- **권한:** `INSTALL.bat` 은 `net session` 으로 관리자 여부를 확인하고, 필요 시 UAC 승격을 제안합니다.
